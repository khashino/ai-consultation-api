How to test this widget

Click the floating chat button in the bottom-right corner. Ask a question that exists in your uploaded documents. The widget will call your backend and show only the assistant answer.

  1. Make sure the backend is running with uvicorn main:app --reload.
  2. Open the admin console at /app and upload at least one knowledge document.
  3. Return to this page and click the floating chatbot button.
  4. Ask a question related to the uploaded knowledge base.

Embed code

To embed this widget into another page, add this configuration and script tag before the closing </body> tag. 

<script>
  window.AI_CHATBOT_API_BASE = "http://127.0.0.1:8000";
  window.AI_CHATBOT_TITLE = "RAG Assistant";
  window.AI_CHATBOT_SUBTITLE = "Answers from the local knowledge base";
  window.AI_CHATBOT_TOPIC = "general";
  window.AI_CHATBOT_USER_COUNTRY = null;
  window.AI_CHATBOT_TARGET_COUNTRY = null;
</script>

<script src="http://127.0.0.1:8000/static/chatbot-widget.js"></script>

What this widget does

Frontend behavior

It creates a floating chat button, opens a compact chat window, stores a session ID in localStorage, and sends messages to your backend.

Backend behavior

It calls POST /chat. Your backend loads recent chat history from SQLite, retrieves relevant context from ChromaDB, sends a prompt to Ollama, and returns the final answer.

Important note

The widget is public by design. Admin features such as document upload, debug, model switching, and chat history management remain protected behind login.
